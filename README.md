# dev-journey-2024

**CS50 and the year computer science became serious – 2024**

2024 was the year I decided to move from using code as a tool to understanding how programming and computer science really work.  
I enrolled in Harvard’s CS50 — my first formal step into the field — and it completely shifted how I approached problems.

Coming from a background where mathematics was not my strongest subject in high school, CS50 showed me that strong problem-solving, logical thinking, and consistent practice can bridge any gap.  
The course’s problem sets forced me to think algorithmically, structure code carefully, and debug rigorously — skills I had only touched lightly before.

### What’s inside

- **CS50 Problem Sets**  
  Solutions to most weeks in C (early psets) and Python (later weeks), including classics like Mario, Credit, Readability, Tideman, Recover, Filter, and more.  
  These exercises built my foundation in low-level programming, memory management, data structures, and algorithmic thinking.

- **A* Pathfinding (personal project)**  
  After CS50, I implemented A* from scratch to solve maze and grid pathfinding problems.  
  This was my way of taking the algorithmic concepts I learned and applying them independently to a classic challenge.

- **Web Track experiments**  
  Early explorations with HTML, CSS, and JavaScript from CS50’s web track — small interactive pages that helped me connect algorithms with real user interfaces.

- **CS50 Final Project (recreated)**  
  My capstone project submitted via me50 (private repo: github.com/me50/Sogonna/tree/cs50/problems/2024/x/project).  
  This is a personal recreation after losing local files. Original is private per CS50 policy.

### Technologies & languages

- C (low-level programming and memory)
- Python (data handling, algorithms, scripting)
- HTML, CSS, JavaScript (web fundamentals)
- Algorithmic thinking (sorting, searching, graphs, recursion)

CS50 was not just a course — it was where I learned that rigorous problem-solving and persistence can turn anyone into a capable programmer, regardless of starting point.  
These projects are not perfect; they contain beginner choices and honest mistakes.  
But they are real, and they mark the moment I committed to computer science as a serious path.

This year built the foundation for everything that followed — front-end development at Hexa and beyond.
